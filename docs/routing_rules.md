# 路由规则

## 一、总原则
先识别任务，再识别资产，再选框架。
禁止一上来就套用固定模板。

## 二、任务路由

### 1. 个股分析
触发词：
- 这只股票能买吗
- 分析XX
- 现在买XX值不值

默认框架组合：
- 巴菲特
- 芒格
- 马克斯

补充框架：
- 成长股 -> 林奇 / ARK
- 低估值成熟股 -> 格雷厄姆 / 格林布拉特
- 金融股 -> 弱化格雷厄姆数，强化资产质量与周期
- 周期股 -> 强化马克斯周期与风险框架

### 2. ETF / 基金分析
触发词：
- 这只ETF能买吗
- 这只基金适合长期持有吗
- 帮我看看基金结构

默认框架组合：
- 基金结构分析
- 风险暴露分析
- 组合角色判断

### 3. 持仓复盘
触发词：
- 我持有XX怎么办
- 成本多少，现在该卖吗
- 这只票要不要补仓

默认框架组合：
- 原始逻辑验证
- 当前估值与风险
- 机会成本审查
- 仓位建议

### 4. 组合构建
触发词：
- 给我配一个组合
- 我有30万怎么投
- 稳健ETF组合怎么配

默认框架组合：
- Bogle
- Dalio
- 现金流与风险承受度适配

### 5. 股票池/基金池筛选
触发词：
- 帮我筛选
- 找几只符合条件的股票
- 给我一组ETF候选

默认框架组合：
- 条件过滤
- 风险排序
- 分层推荐

## 三、资产类型路由

### 股票
- 成熟消费 / 平台 / 龙头制造：巴菲特优先
- 小盘成长：林奇 + 马克斯
- 高估高增速科技：ARK + 马克斯 + 芒格
- 周期资源：马克斯优先
- 银行保险：资产质量、负债结构、资本充足率优先

### 基金
- 宽基ETF：核心配置视角
- 行业ETF：卫星配置或阶段交易视角
- 主动基金：看经理、漂移、费率、回撤
- 黄金/商品ETF：对冲或战术配置视角

## 四、结论路由
### 可直接买入
需要同时满足：
- 逻辑清晰
- 风险可承受
- 估值不过分
- 数据完整度至少B

### 只能观察
出现以下任一情况：
- 公司不错但价格贵
- 基金结构还行但位置拥挤
- 数据不完整
- 叙事强于事实

### 直接放弃
出现以下任一情况：
- 超出能力圈
- 负债风险大
- 现金流差
- 明显靠故事支撑
- 产品/行业结构恶化

## 五、仓位建议原则
- 高不确定高波动：轻仓试错或不做
- 确定性高但估值一般：分批
- 核心ETF：可作为长期定投或底仓
- 单一个股：原则上不建议仓位失控

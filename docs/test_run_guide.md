# 试跑指南

## 一、先测什么
先不要一上来测最复杂场景。
建议按下面顺序试跑：

1. 只输入一只股票名称
2. 输入一只ETF名称
3. 输入一只持仓及成本
4. 输入一个组合需求

## 二、建议测试指令

### 个股
分析腾讯，给我明确建议。

### ETF
分析QQQ，判断它适不适合长期持有。

### 持仓
我持有斯瑞新材，成本45元，仓位15%，原始逻辑是看中成长性。现在怎么办？

### 组合
我有30万，35岁，优先稳健增值，给我做一个ETF组合。

## 三、试跑时看什么

### 1. 有没有先识别资产类型
如果连股票、ETF、持仓、组合都不先区分，这个 Skill 结构就是松的。

### 2. 有没有先判断数据完整度
如果数据不足却直接给高置信买入，就是有问题。

### 3. 有没有给明确结论
不能只讲道理，不落到动作。

### 4. 有没有写反方观点
没有反方观点，说明它容易顺着用户走。

### 5. 有没有说明置信边界
不说明哪些是硬判断、哪些是条件判断，说明输出不够克制。

## 四、合格输出的最低标准
- 资产识别正确
- 结论明确
- 风险说到点上
- 不伪造数据
- 不夸张自信

## 五、下一步优化方向
试跑后重点看三类问题：
- 路由错了
- 结论太软
- 在缺数据时装懂
